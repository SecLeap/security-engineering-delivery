# 安全服务项目交付（技术侧）知识库

面向安全服务项目的技术交付知识库。当前阶段只建立目录、能力地图与待办清单；所有结论、脚本、样例数据和客户敏感信息均应在后续按授权范围补充。

## 使用方式

1. 从 [能力阶梯/从基础到高阶.md](能力阶梯/从基础到高阶.md) 确定当前能力阶段。
2. 项目启动时先使用 [交付通用能力/项目全生命周期.md](交付通用能力/项目全生命周期.md) 和对应服务项清单。
3. 交付过程中将经过脱敏的经验沉淀到对应专题、场景或服务目录，并维护各文件中的待办项。
4. 所有实战操作必须在合同、书面授权和变更窗口内执行；不得在知识库中保存密码、密钥、客户数据或可直接用于未授权攻击的内容。

## 建设原则

- 先有可执行的清单，再补充方法、示例和自动化。
- 每份服务知识至少覆盖：授权边界、输入、实施、证据、风险分级、交付物、验收与复盘。
- 以可复用、可审计、可脱敏为准；工具和攻击验证均只记录合规、安全的使用方式。
